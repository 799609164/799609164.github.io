MyBatis 框架

1 简介

1.1 概念

  • ORM（Object/Relational Mapping），即对象关系映射，实现面向对象的编程语言到关系数据库的映射

  • ORM 工具的唯一作用：把持久化对象的保存、修改、删除等操作，转换为对数据库的操作

  • ORM 基本映射关系

    • 数据表——类
    • 数据表的行——对象（实例）
    • 数据表的列（字段）——对象的属性

  • MyBatis 的主要思想：将程序中的大量 SQL 语句抽取出来，配置在配置文件中，以实现 SQL 的灵活配置

  • MyBatis 可以对配置和原生 Map 使用简单的 XML 或注解，将接口和 Java 的 POJO 类映射成数据库中的记录

  • MyBatis 并不完全是一种 ORM 框架，它的设计思想和 ORM 相似，只是它允许直接编写 SQL 语句，使得数据库访问更加灵活；准确地说，MyBatis 提供了一种“半自动化”的 ORM 实现，是一种 “SQL Mapping” 框架

1.2 功能架构

  • API 接口层
    • 提供给外部使用的接口 API，接口层接收到调用请求会调用数据处理层来完成具体的数据处理
    • 数据查询接口/数据新增接口/数据更新接口/数据删除接口/获取配置接口
  • 数据处理层
    • 负责具体的 SQL 查找、解析、执行以及执行结果映射处理，实现根据调用请求完成一次数据库操作
    • 参数映射：参数映射配置/参数映射解析/参数类型解析
    • SQL解析：SQL 获取/SQL 解析/动态 SQL
    • SQL执行：SimpleExecutor/BatchExecutor/ReuseExecutor
    • 结果映射：结果映射配置/结果类型转换/结果数据拷贝
  • 基础支撑层
    • 负责最基本的功能支撑
    • 连接管理/事务管理/配置加载/缓存处理

1.3 框架架构

  • 加载配置
    • MyBatis 应用程序根据 XML 配置文件加载运行环境，创建 SqlSessionFactory ，将 SQL 的配置信息加载成为一个个 MappedStatement 对象，包括传入参数映射配置、执行的 SQL 语句、结果映射配置，存储在内存中
  • SQL 解析
    • 当接口层接收到调用请求时，会接收到传入 SQL 的 ID 和传入对象（Map/JavaBean/基本数据类型），MyBaits 根据 SQL 的 ID 找到对应的 MappedStatement，再根据传入的参数对象对 MappendStatement进行解析，解析后得到最终要执行的 SQL 语句和参数
  • SQL 执行
    • SqlSession 将最终得到的 SQL 和参数拿到数据库进行执行，得到操作数据库的结果
  • 结果映射
    • 将操作数据库的结果按照映射的配置进行转换，转换为 HashMap/JavaBean/基本数据类型，返回结果后关闭 SqlSession

1.4 其他

  • 优点

    • 简单小巧易于上手，方便浏览修改 SQL 语句
    • 解除 SQL 与程序代码的耦合
    • 提供映射标签，支持对象与数据库 ORM 字段关系映射
    • 提供 xml 标签，支持编写动态 SQL

  • 与 JDBC、Hibernate 的区别

    • MyBatis 在 JDBC 的基础上做了，OO化、封装事务管理接口等，依然要通过 JDBC 完成数据查询、更新等操作
    • MyBatis 与 Hibernate 都屏蔽了 JDBC API 的底层访问细节
    • Hibernate 是全自动的 ORM 映射工具，可以自动生成 SQL 语句；MyBatis需要在 xml 配置文件中写 SQL 语句
    • 由于 Hibernate 可以自动生成 SQL 语句，在进行复杂查询时，Hibernate 实现要比 MyBatis 复杂
