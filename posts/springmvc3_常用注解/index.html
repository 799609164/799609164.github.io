3 常用注解

3.1 @Controller

  • 控制器 Controller 负责处理由 DispatcherServlet 分发的请求，把用户请求的数据经过业务处理层处理后封装成一个 Model，再把 Model 返回给对应的 View 进行展示

  • 不使用注解

2

<!-- 在 springmvc-config.xml 中添加 Controller 类的 Bean 对象 -->
<bean class = "com.xxx.springmvc.controller.UserController"/>

  • 使用注解

    • 使用 @Controller 注解的类不再需要继承特定的基类或实现特定的接口，并且可以同时处理多个请求动作

// 2_入门程序中 com.xxx.springmvc.controller.UserController.java
// Spring 会使用扫描机制查找所有基于注解的控制器类
@Controller
public class UserController {

	/**
		* RequestMapping 用来映射一个请求和请求的方法
		* value = "/register" 表示请求由 register 方法进行处理
		*/
	@RequestMapping(value = "/register")
	public String Register(User user, Model model) {
		// user:视图层传给控制层的表单对象；model：控制层返回给视图层的对象
		// 在 model 中添加一个名为 "user" 的 user 对象
		model.addAttribute("user", user);
		// 返回一个字符串 "success" 作为视图名称
		return "success";
	}
}

3.2 @ RequestMapping

  • 用于处理请求地址映射，指示 Spring 用哪个类或方法来处理请求动作；作用在类或方法上

// 当 @RequestMapping 用于类上时，表示类中的所有响应方法都被映射到 value 属性所指示的路径下
// url=http://localhost:8080/user/register
@Controller
@RequestMapping(value = "/user")
public class UserController {
	@RequestMapping(value = "/register")
	public String Register() {
		return "register";
	}
}

  • 常用属性

    • value：指定请求的实际地址，value 属性将 URL 映射到方法上；如果是唯一的属性，value 属性名可省略
    • method：指定请求的 method 类型，即该方法可以处理的 HTTP 请求方式，如 GET、POST 等；如果未指定 method，则可以处理任意类型的 HTTP 请求
    • consumes：指定处理请求的提交内容类型（Content-Type），如 application/json, text/html
    • produces：指定返回的内容类型，仅当 request 请求头中的（Accept）类型中包含该指定类型才返回
    • params：指定 request 中必须包含某些参数值，才让该方法处理请求
    • headers：指定 request 中必须包含某些指定的 header 值，才让该方法处理请求

  • 请求处理方法的返回类型

    • Model

      • Spring MVC 内部使用 Model 接口存储模型数据，功能类似于 Java 中的 Map，而 ModelMap 接口是实现了 Model 接口；因此，Model 和 ModelMap 的用法相近

    • ModelAndView

      • 既包含模型数据信息又包含视图信息，Spring MVC 将包含的视图对模型数据进行渲染，模型数据可以看做 Java 中的 Map

// 1. Model
@RequestMapping(value = "/register")
// 可以将参数中的 Model model 改为 ModelMap modelMap
public String Register(User user,Model model) {
	// jsp 页面传入的 form 表单
	// 将 User 对象添加到 Model 中
	model.addAttribute("user", user);
	return "success";
}

// 2. ModelAndView
@RequestMapping(value = "/register")
public ModelAndView Register(User user,ModelAndView mv) {
	// jsp 页面传入的 form 表单
	// 将 User 对象添加到 ModelAndView 的 Model 中
	mv.addObject("user", user);
	// 设置返回视图的名称
	mv.setViewName("success");
	return mv;
}

3.3 参数绑定

  • @RequestParam

    • 用于在 Spring MVC 后台控制层获取参数，类似一种是 request.getParameter("name")
    • name：指定请求头绑定的名称
    • value：name 的别名
    • required：通过 boolean 指定参数是否必须绑定
    • defaultValue：设置默认值

  • @SessionAttributes

    • 允许有选择地指定 Model 中的哪些属性需要转存到 HttpSession 对象中；只能用于类上，不能用于方法上

@Controller
public class ModelAttributeController {
	
	// 注解 @ModelAttribute("user")，表示 userModel 方法会先于 login 方法调用
	// model 属性名称是 value 指 "user"，model 属性对象是方法的返回值
	@ModelAttribute("user")
	public User userModel(
		@RequestParam("username") String username, @RequestParam("password") String password){

		User user = new User();
		user.setUsername(username);
		user.setPassword(password);
		return user;
	}
	
	// @ModelAttribute("user")，表示参数 user 的值是 userModel 方法中的 model 属性
	@RequestMapping(value = "/login")
	public String login(@ModelAttribute("user") User user{
		return "success";
	}
}
